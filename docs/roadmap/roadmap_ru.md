# Roadmap ReDrive

> Этот roadmap описывает развитие ReDrive от текущего состояния до полноценного MVP и дальнейшего расширения проекта.
> 
> Roadmap не является жёстким обещанием сроков. Он нужен, чтобы понимать направление разработки, приоритеты и порядок реализации задач.

---

## Легенда

|Метка|Значение|
|---|---|
|`[x]`|Задача выполнена|
|`[ ]`|Задача запланирована|
|`MVP`|Обязательно для первой полноценной версии|
|`После MVP`|Можно делать после первого релиза|
|`Future`|Идея на будущее|

# Этап 1. Приведение OBD-архитектуры в порядок

**Статус:** MVP  
**Цель:** убрать лишнюю ответственность из `ObdProvider` и подготовить проект к Wi-Fi, USB, ошибкам, Garage и будущим расширениям.

`ObdProvider` не должен содержать всю бизнес-логику проекта. Его задача — отдавать готовое состояние в UI, а не самостоятельно заниматься ELM-командами, парсингом PID и диагностикой.

## Разделение ответственности

- [ ] Разделить текущий `ObdProvider` на более мелкие части
- [ ] Оставить в `ObdProvider` только состояние, нужное для UI
- [ ] Вынести отправку и ожидание ELM-команд в отдельный сервис
- [ ] Вынести парсинг PID-ответов в отдельный класс
- [ ] Вынести polling-логику в отдельный контроллер или сервис
- [ ] Вынести обработку ошибок OBD/ELM в отдельные типы ошибок
- [ ] Убрать дублирование логики между Bluetooth, demo mode и будущими типами подключения
    
## ELM327 core

- [ ] Создать `ElmClient`
- [ ] Создать `ElmCommandQueue`
- [ ] Гарантировать выполнение только одной команды за раз
- [ ] Сделать единое место, где к командам добавляется `\r`
- [ ] Добавить корректное ожидание prompt `>`
- [ ] Добавить timeout для команд
- [ ] Обрабатывать ответы `NO DATA`, `STOPPED`, `SEARCHING`, `UNABLE TO CONNECT`
- [ ] Добавить raw-логирование команд и ответов для отладки
- [ ] Добавить безопасную очистку буфера при disconnect/reconnect
    
## PID core

- [ ] Создать модель `PidDefinition`
- [ ] Создать `PidRegistry`
- [ ] Создать `PidDecoder`
- [ ] Перенести стандартные PID в отдельный список/registry
- [ ] Добавить unit-тесты для формул базовых PID
- [ ] Добавить поддержку разных единиц измерения
- [ ] Добавить категории PID: engine, vehicle, fuel, temperature, diagnostic

```text
UI
  ↓
Provider
  ↓
ObdSession / ObdService
  ↓
ElmClient
  ↓
ObdConnection / Transport
  ↓
Bluetooth / Wi-Fi / USB / Mock (demo)
```

- [ ] Добавить unit-тесты для `ElmCommandQueue` (очередность, таймауты)
- [ ] Добавить unit-тесты для парсинга сырых ответов ELM327 (OK, NO DATA, SEARCHING)
- [ ] Добавить unit-тесты для формул вычисления PID
- [ ] Добавить тесты на корректную очистку буфера при переподключении
